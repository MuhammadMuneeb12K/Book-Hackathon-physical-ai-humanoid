Getting Started - Setting Up Your Environment

This section provides essential guidance for setting up your development environment, ensuring you have all the necessary tools and configurations to follow along with the book's practical exercises. A well-prepared environment is crucial for a smooth learning experience in the complex domain of humanoid robotics.

Goal​

The goal of this "Getting Started" guide is to establish a robust and consistent development environment that supports all the modules and hands-on labs presented in this book. Upon completion, you will have a functional setup for ROS 2 development, simulation, and AI model integration.

Learning Objectives​

  • Understand the recommended operating system and hardware requirements.
  • Successfully install and configure Ubuntu 22.04 LTS (Jammy Jellyfish).
  • Install NVIDIA GPU drivers and CUDA Toolkit for accelerated computing.
  • Set up ROS 2 Humble Hawksbill, including all necessary development tools.
  • Configure Python environments and package managers (pip, conda).
  • Understand and prepare for potential hardware-specific dependencies (e.g., NVIDIA Jetson, RTX GPUs).

Prerequisites​

  • A dedicated computer (desktop or laptop) capable of running Ubuntu.
  • For advanced simulation (Isaac Sim) and AI tasks, an NVIDIA GPU (RTX series recommended) is highly advisable.
  • Basic familiarity with Linux command-line operations.
  • Stable internet connection for downloading software packages.

Key Concepts​

  • Operating System (OS) Choice: The importance of a stable Linux distribution for robotics development.
  • Package Management: Using apt and pip for software installation.
  • GPU Acceleration: Understanding the role of NVIDIA drivers and CUDA in AI and simulation.
  • ROS 2 Distributions: The concept of long-term support (LTS) distributions like Humble Hawksbill.
  • Python Virtual Environments: Best practices for managing Python dependencies.
  • Containerization (Optional but Recommended): Introduction to Docker for consistent environments (briefly mentioned for advanced users).

Tools​

  • Operating System: Ubuntu 22.04 LTS (Jammy Jellyfish).
  • ROS 2 Distribution: Humble Hawksbill.
  • Python: Version 3.10 (default with Ubuntu 22.04).
  • NVIDIA Drivers & CUDA Toolkit: Specific versions compatible with your GPU and Isaac Sim.
  • apt: Ubuntu's default package manager.
  • pip: Python's package installer.
  • git: Version control system.
  • Code Editor: Visual Studio Code (recommended) with relevant extensions (e.g., Python, ROS).

Hands-on Labs​

Lab 1: Verify Ubuntu and Basic Utilities​

  1. Objective: Confirm a successful Ubuntu installation and basic command-line functionality.
  2. Instructions:
    • Open a terminal (Ctrl+Alt+T).
    • Run lsb_release -a to verify Ubuntu 22.04.
    • Run python3 --version to check Python installation (should be 3.10.x).
    • Run sudo apt update && sudo apt upgrade -y to update your system.
    • Install git if not already present: sudo apt install git -y.
    • Verify git installation: git --version.

Lab 2: Confirm NVIDIA Driver and CUDA Installation​

  1. Objective: Ensure NVIDIA GPU drivers and CUDA Toolkit are correctly installed and detected.
  2. Instructions:
    • Open a terminal.
    • Run nvidia-smi to verify driver installation and GPU status. Note the CUDA version reported by nvidia-smi (this is the driver-API compatibility, not the toolkit version).
    • Run nvcc --version to confirm the CUDA Toolkit installation and its version. This version should ideally match or be compatible with the CUDA version required by Isaac Sim or other deep learning frameworks you intend to use.
    • (Optional) Run a CUDA sample to verify functionality: cuda-install-samples-$(nvcc -V | grep -oP 'V\d+\.\d+' | cut -c 2-)/NVIDIA_CUDA-$(nvcc -V | grep -oP 'V\d+\.\d+' | cut -c 2-)/bin/x86_64/linux/release/deviceQuery. Look for "Result = PASS".

Lab 3: Validate ROS 2 Humble Setup​

  1. Objective: Verify that ROS 2 Humble Hawksbill is correctly installed and configured.
  2. Instructions:
    • Open a new terminal.
    • Source the ROS 2 setup script: source /opt/ros/humble/setup.bash.
    • Run a ROS 2 talker node: ros2 run demo_nodes_cpp talker.
    • Open a second terminal.
    • Source the ROS 2 setup script: source /opt/ros/humble/setup.bash.
    • Run a ROS 2 listener node: ros2 run demo_nodes_py listener.
    • Observe messages being exchanged between the two terminals.

Expected Output​

  • A fully configured Ubuntu 22.04 system with NVIDIA GPU drivers, CUDA Toolkit, and ROS 2 Humble Hawksbill installed.
  • Ability to run basic ROS 2 example nodes (talker/listener).
  • A functional Python environment ready for robotics and AI development.

Real-world Applications​

  • Robotics Development Labs: Standardizing development environments for student projects and research.
  • Industrial Automation: Setting up consistent deployment targets for robot control software.
  • AI/Machine Learning Engineering: Ensuring reproducible research environments for model training and deployment.
  • Embedded Systems Development: Preparing cross-compilation environments for robotics hardware.

Edge Cases​

  • GPU Compatibility Issues: Older NVIDIA GPUs might not support the latest CUDA versions required by Isaac Sim.
  • ROS 2 Dependency Conflicts: Other installed software might conflict with ROS 2 packages.
  • Network Proxy Settings: Installation issues in environments with strict network proxies.
  • Disk Space Limitations: Insufficient disk space for installing all necessary tools and datasets.
